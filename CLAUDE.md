## What this is

`floridayvine` is a Typer CLI that ingests Floriday trade information into Serra
Vine and persists it to MongoDB. Code is under `src/floridayvine/`, with the
Floriday integration in `src/floridayvine/floriday/`. Current and planned work
is tracked in [`work/doing.md`](work/doing.md) and
[`work/backlog.md`](work/backlog.md).

## Floriday API client

Trade data comes from the `floriday-supplier-client` PyPI package (pinned in
`pyproject.toml`). The client validates `FLORIDAY_BASE_URL` against the API
version it was generated for, so the two must stay in lockstep.

When upgrading the client, bump the version string across env files, CI
workflows and docs in one pass — don't hand-edit them individually:

```
make set-api-version API_VERSION=2026v1   # run with no arg to list current refs
```

## Conventions

- **Releases:** `make release` does the whole thing (regenerate docs, bump the
  CHANGELOG `[Unreleased]` heading, tag, push). Don't run the git steps by hand.
  See CONTRIBUTING.md → *Release and Deployment*.
- **CLI docs** (`docs/cli_documentation.md`) are generated by `make
  documentation` — don't edit them by hand.
- **Before committing:** run `make quality` (flake8 + black + tests).
- `.claude/settings.local.json` is machine-local and gitignored — never commit
  it.
